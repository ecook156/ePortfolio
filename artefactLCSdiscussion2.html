Collaborative Discussion 1

Identify and discuss two security technologies and the context in which they can be employed.

Initial Post

When it comes to designing secure software, Samar & Lai (1996) argue that authentication technologies require special attention because it forms a critical component of a secure system. “Passwords are the most widespread form of user authentication on the web today” (Reese et al, 2019) but these can be manipulated through social engineering techniques, weak passwords and password sniffing (Reid, 1996). The IEEE recommends using an authentication method that cannot be compromised (Ramakrishnan & Gunter, 2017). Two technologies that aim to strengthen authentication are two-factor and IP based authentication.

IP based authentication is a method that identifies a user by the IP address associated with a client. It performs validation of certain IPs against a list of whitelisted addresses and grants access when there is a match. Users gain access to a system based on their IP address which ensures ease of us, an important characteristic of authentication and not having to remember credentials helps support this (Samar & Lai, 1996). Implementing IP based authentication in an organisation is relatively quick and cheap making it a good choice for large companies.

However, this can be bypassed by taking physical control of a client and therefore having access to anything that IP address can access, or by spoofing the IP address through packets to make it appear correct even though it has originated somewhere else (Reid, 1996). While physical access to a client can be protected through security measures such as locks, there are many filtering products that can help to detect IP spoofing.

Two-factor (or multi factor) authentication is another technology that can be used to mitigate the risk of humans using weak passwords, by requiring more than one piece of evidence to gain access (Brookshear & Brylow, 2020). It requires a user to authenticate using something they know and something they either have or are. This usually includes a biometric “identifying people by measuring some aspect of individual anatomy or physiology” (Anderson, 2020) or a smartcard or token on a mobile device. “A common multi-factor authenticator is an ATM card, which combines a token with a secret (PIN)” (Beng Jin et al, 2004). It is proven that humans are not very good at remembering secure passwords (Brookshear & Brylow 2020) therefore the use of biometrics as a second authentication method makes it difficult for the user to forget and also harder for tokens to be faked (Reid,1996).

While two-factor authentication methods are undoubtedly more secure than a single factor, it comes at extra cost for service providers and requires users to take additional steps during the authentication process (Cristofaro et al, 2013). Studies have shown that users become frustrated with not receiving their second factor passcode straight away (Reese et al 2019) which could discourage the use of 2FA.

Summary Post

Throughout the collaborative discussion it has become clear that there are many different security technologies available covering both proactive measures, to secure both software and networks, and reactive to detect malicious activity that occurs (Venter & Eloff, 2003). My initial post focused on authentication technologies that aim to secure software by granting or denying permission to a system based on various criteria (Okafor-Tolofari, 2022). Meadows (2022) states that in the current digital world, having multiple forms of authentication can make it more difficult to fake an identity.

Single factor authentication cannot be relied upon to provide adequate protection to security threats (Ometov et al, 2018). Many users tend to reuse passwords, but with the introduction of authentication technologies, even if a password is stolen, it is unlikely the attacker will have the second part of the authentication (Reese et al, 2019).

While both MFA and IP based authentication add a layer of security, they can be manipulated through social engineering or spoofing (Reid, 1996) therefore using these technologies in combination with other methods would strengthen them. Pal (2021) argues that physical security technologies, such as restriction of physical access, are fundamental to security controls. This is the case with IP based authentication where taking physical control is its downfall. Usability can become a challenge when enforcing MFA as it can have a negative effect on task efficiency (Ometov et al, 2018) and Angelides (2021) points out that MFA will only provide the maximum benefit once users start to implement it in all areas that need protection.

To strengthen authentication technologies, it is key to incorporate other layers of security in the form of network security, monitoring and intrusion detection. Havryliuk (2021) argues the importance of a Security Operations Centre (SOC) in identifying anomalies in traffic, this could help to detect malicious entry into a system if the authentication methods were breached. Thus, combining technologies that combat intrusion prevention with detection will create a layered approach to security.
