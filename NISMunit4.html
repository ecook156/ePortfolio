Collaborative Learning Discussion 2

As part of Group 2 we were assigned the website blog https://allmytype.co.uk. On our first weekly meeting we discussed the activity and Samuel demonstrated some examples of tools we could use to perform the scans. We decided to each have a go at finding the information before our scheduled meeting the following week which we could then use to discuss the findings and any issues we may have encountered. Having each member of the team perform their own scans using their own choice of tools meant we could be confident with the results which we all found to be the same.

Initial Post

As I am new to this kind of practical activity, I chose to use simple online tools as well as the Traceroute command on Mac as described by a2hosting.coms knowledgebase. The Traceroute scan took me 10 hops to get to the website with an average delay of 29.8ms. In order to provide more accurate results, I should have performed this multiple times to see if the results differed and assess the reliability of the results.

I used who.is to search for the registered contact of the domain which told me that the registrar was eNom LLC URL: http://www.enom.com but there was no name or address available associated with the 3rd party. The information on this website is provided by Nominet UK which is the central register for .uk domain names. I then used DNS Spy to run a report on the domain which provided all of the remaining information required such as IP address, MX record and main name servers.

From the activity I found that all of the 4 name servers were hosted by the same provider (Cloudflaranet) based in the US.
